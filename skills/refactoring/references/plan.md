## 分析方針

本分析は以下の原則に基づいて実施：

- **凝集度**: 機能的凝集を目指す（単一の責務のみを持つ）
- **結合度**: メッセージ結合・データ結合を目指す（必要な値だけ渡す）

### 凝集度の分類（1=最良 → 7=最悪）

1. **機能的凝集**: 単一の機能だけを実現
2. 逐次的凝集: 処理間で値の受け渡しがある
3. 通信的凝集: 同じデータを使う処理群
4. 手順的凝集: 特定の順序に意味がある処理
5. 時間的凝集: 特定の時間に実行される処理
6. **論理的凝集**: フラグなどで処理を切り替える
7. **偶発的凝集**: 関連性のない処理が1つの関数にある

### 結合度の分類（1=最良 → 7=最悪）

1. **メッセージ結合**: 値の受け渡しなし
2. **データ結合**: 必要な値だけ渡す
3. **スタンプ結合**: 構造体やクラスを丸ごと渡す
4. **制御結合**: フラグで相手の動作を制御
5. 外部結合: public変数経由
6. 共通結合: グローバル変数共有
7. 内容結合: 他関数の内部を直接変更
